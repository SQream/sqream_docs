DESCRIBE QUERY

The DESCRIBE QUERY command replaces the SHOW_NODE_INFO command. You can use it to display information about query execution for monitoring and troubleshooting purposes.

Note

DESCRIBE commands use CPU to increase usability.

Syntax

The following is the syntax for the DESCRIBE QUERY command:

DESCRIBE QUERY [SESSION ID <session-id>] QUERY ID <query-id>;

Parameters

The following parameters can be used when switching databases with the DESCRIBE QUERY command:

Parameter Name

Parameter Value

Description

Type

SESSION ID

session_id

(Optional) The session ID of the query. If not supplied current session ID is used.

Int

QUERY ID

query-id

The query ID of the user.

Text
